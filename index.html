HTML

  • (!DOCTYPE html) at the top of every HTML file.
  • Then you have (html lang="en") to make the base language clear.
  • The (head) element contains information about the webpage.
    • (meta charset)
    • (meta http-equiv)
    • (meta viewport)
    • (title)
  • The (body) element represents the visible content shown to the user.
    • i.e. (h1-6), (div), (p), (img), etc

CSS

  • selector {proptery: value; property: value;}
  • Single sets of {property: value;} are declarations.
  • "*" rules apply to entire stylesheet.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Rule of Thumb: Start broad, "cascade" down to the more specific.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • add, commit, pull, push
    • git add -A
    • git commit -m "commit notes"
    • git pull origin main
    • git push origin branch-name
    • then switch to GitHub and create PR (pull request) and/or merge

JavaScript

  • JavaScript is an object-oriented programming (OOP) language.
  • js files are executed from the top to the bottom, aka 'control flow'.
  • (script src) goes below (footer), but still in (body).
  • for loop, repeating console.log() per variable:
    • for(var x = 0; x < shapes.length; x++) {code block}
  • /* */ for blocking out sections as comments
  • Arrays got L to R, Top to Bottom, and start numbering with 0.
